단락의 위치와 단락 사이의 간격, 줄 간격 등을 변경할 수 있습니다.

Word 개념정리 디자인 탭/ 레이아웃 탭
설명 : 디자인 탭의 기능들과 단축키에 대해서 정리한다.
4. 디자인
[템플릿] 
 쉽게 교체가능한 이미지와 가짜 텍스트를 포함해 사전 생성된 상태로 시작하는 것으로 템플릿을 참조하여 디자인을 할 수 있도록 하는 기능이다.

[테마] 

 디자인 탭을 클릭하고 타일, 부제목, 본문 텍스트 같은 인기 레이아웃 옵션을 위한 폰트 콜렉션인 테마를 선택할 수 있다.
 또한 테마의 배경 색상, 패턴, 글꼴 등 전체적인 디자인을 변경할 수 있는 옵션이 존재한다.

[워터마크] 
 저작권 보호를 목적으로 많이 사용하는 워터마크는 디자인 탭에서 설정하며, 대부분 회사로고나 문서의 보안에 관련된 내용을 워터마크로 표시한다.

[페이지 색] 
 페이지의 배경이 되는 색을 변경하는 기능이다.

[페이지 테두리] 
 페이지의 테두리를 설정하는 기능으로, 상자,그림자, 3차원 등의 기능을 제공하고 색과 두께와 선의 스타일의 옵션을 선택 가능하다.


5. 레이아웃
[텍스트 방향] 
 텍스트를 가로 또는 세로로 배치할 수 있으며 한 줄에 표시 되거나 여러 줄로 래핑할 수 있다.
 텍스트의 방향을 변경 하고, 텍스트 상자 및 도형의 여백을 변경 하 여 최적의 간격으로 만들거나, 도형의 크기를 조정 하여 텍스트에 더 잘 맞출 수도 있다.

[여백]
 미리 정의된 여백 설정을 지정하거나 선택할 수 있습니다. 마주 보는 페이지의 여백을 설정하고, 문서 제본을 위한 여분의 여백 공간을 허용하고, 여백을 측정하는 방법을 변경할 수 있습니다.

[용지 방향] 
 용지의 방향을 세로 또는 가로로 변경할 수 있습니다.

[단]
 열이 있는 문서에 서식을 지정 하는 경우에는 한 열에서 다른 단으로 텍스트가 자동으로 변경됩니다.

[용지 크기] 
 이미 정의 되어 있는 선택으로 용지의 크기를 변경할 수 있습니다. 또는 기타 용지 크기를 사용하여 직접 원하는 크기로 설정할 수 있습니다.

[원고지]
 눈금 원고지, 밑줄 원고지, 윤곽선 원고지를 선택할 수 있고, 행과 열을 변경할 수 있습니다.

[단락] 
 줄 간격을 설정 하여 문서에서 텍스트 줄 사이의 세로 간격을 조정할 수 있습니다. 또는 단락 앞뒤 간격을 설정하여 문서에서 단락 사이의 세로 간격을 변경할 수 있습니다. 텍스트 줄을 함께 유지 하거나 페이지에서 단락을 함께 유지 하도록 선택할 수도 있습니다. 또는 들여쓰기를 사용할 수 있습니다.

[정렬] 
 그림, 도형, 텍스트 상자 등의 개체들을 빠르게 정렬하게 해주는 기능입니다.

스타일을 정할 수 있어 다양한 형태로 입력할 수 있다. APA가 흔하게 사용된다.

Word 개념정리 검토 탭 / 보기 탭
설명 : 검토 탭/보기 탭의 기능들과 단축키에 대해서 정리한다.
8. 검토
[언어 교정]
-맞춤법 및 문법 검사: 문제가 있는 단어나 문장을 찾아 올바르게 고칠 수 있다.

[접근성]

-번역: 자신이 원하는 언어로 번역할 수 있다.

[언어]
-번역: 원하는 언어로 번역할 수 있다.
-언어: 원하는 언어를 기본 언어로 설정할 수 있고 자동으로 사용할 언어를 설정할 수 있다.

-메모 표시: 문서의 모든 메모를 확인할 수  있다.

[추적]
 -변경 내용 추적: 사용자가 어떤 내용을 변경했는지 표시하는 기능이다. 빨간색으로 글자가 그어져 있다.

[변경 내용]

[비교]

[보호]
 -편집을 제한하여 문서를 보호한다.

Ⅰ- 2. Hwp
한글 개념정리
설명 : 워드에는 없는 한글의 기능, 워드와 한글의 차이점에 대해 기술한다.
워드와 한글은 유사해 워드와는 다른 기능을 중심으로 학습

1.	보안
[문서 암호 설정]
파일에 문서 암호를 지정하여 문서 암호를 모르면 누구도 그 파일 내용을 열어 볼 수 없게 합니다. 문서 암호를 정확하게 입력할 때만 문서 파일을 열어 내용을 볼 수 있습니다. 또한 변경 및 해제를 통해 문서에 이미 설정된 문서 암호를 다른 암호로 바꾸거나 해제합니다.

[공인 인증서로 문서 암호화]
 공인 인증서를 통하여 기밀문서의 정보 유출을 막을 수 있고 개인의 문서 보안성을 강화할 수 있습니다.

[공인 인증서로 전자 서명]
 인증서의 공개 키 암호 방식을 이용하여 문서에 디지털 서명을 하는 기능입니다. 문서를 암호화하는 것이 아니므로 제 3자가 문서 내용을 열람하는 데에는 제한 사항이 없고, 작성자가 그 문서를 작성하였다는 사실과 송수신 과정에서 위조,변조 되지 않았다는 사실을 증명합니다.

[유효성 검사]
 공인 인증서로 전자 서명한 문서에는 서명자의 공개 키가 포함되어 있으므로, 공인 인증서로 전자 서명의 유효성을 검증할 수 있습니다.

[배포용 문서로 저장, 편집]
 수정을 불가능한 문서로 만들어 내용을 변경하지 못하도록 만들어 줍니다. 또한 입력된 암호를 통해 문서 속성은 그대로 유지하고 문서를 편집 상태로 만듭니다.

[보안 문서로 저장]
 공인 인증서를 인증 매체로 사용하여 보안 문서로 저장할 수 있습니다. 보안 문서로 저장은 공개 키(PKI) 기반의 암호화 솔루션을 사용하여 문서를 암호화하고 또 필요에 따라 암호가 설정된 문서를 볼 수 있도록 하는 기능입니다. 

[개인 정보 보호]
 문서 일부분을 암호를 걸어 다른 문자로 변경할 수 있어 문서의 보안성을 강화할 수 있습니다. 개인 정보 보안 기능은 문서 전체가 아닌 사용자가 지정한 문서의 일부분만 기밀 처리를 할 때 사용하면 편리합니다.

2.	검토
[교정 부호]
 
맞춤법, 띄어쓰기, 활자 크기, 문장 부호, 줄 바꿈, 오자 등을 바로잡기 위해 사용자가 교정 부호를 삽입할 수 있습니다.

[문서 이력 관리]
 
현재 문서에서 사용자가 원하는 시점 또는 저장할 때에 버전을 삽입합니다. 버전을 2개 선택해서 비교가 가능합니다.

[변경 내용 관리]
 
변경한 내용을 추적하여 수정된 내용을 원본 내용과 구분하여 쉽게 확인이 가능합니다. 안내문 표시를 설정하면 추가 삭제 수정한 내용이 안내문에 표시되고 변경 내용 보기는 삽입하거나 삭제한 내용, 서식 등의 변경 내용을 표시해줍니다.

[원본/최종본]
 
문서의 원본과 최종본 관리가 가능하고 두 버전의 내용을 비교 확인이 가능합니다.

4.	디자인
[테마]
 
 
 테마에서는 색, 배경, 글꼴 스타일 및 개체 틀 배치를 포함하여 다양한 디자인 테마를 제공합니다. 미리 디자인된 테마를 사용하여 프레젠테이션의 전반적인 모양을 빠르게 변경할 수 있습니다. 또한 테마에 따라 적용을 다르게하여 슬라이드 전체 또는 일부의 색을 변경할 수 있습니다.

[슬라이드 크기]
 
기본적으로 4:3 표준 또는 16:9의 와이드스크린의 비율을 사용하거나 사용자 지정 슬라이드 크기를 선택합니다.

[배경 서식]
배경을 단색, 그라데이션, 그림, 질감, 패턴으로 채울 수 있으며, 투명도와 밝기 등을 조절하여 배경서식을 변경할 수 있습니다.

[디자인 아이디어]
 
하나 이상의 그림, 항목 목록 또는 날짜 목록을 삽입하고 디자인 아이디어를 클릭하면 사용자에게 여러가지 디자인이 추천되고, 원하는 디자인을 선택합니다.




5.	전환
[미리보기]
 
 화면 전환에 사용된 효과를 미리볼 수 있습니다.

[슬라이드 화면 전환]
 
 슬라이드 전환은 프레젠테이션 중에 한 슬라이드에서 다른 슬라이드로 이동할 때 발생하는 시각적 효과입니다. 전환 효과의 속도를 제어하고, 소리를 추가하고, 모양을 사용자 지정할 수 있습니다.

[타이밍]
 
전환 기간을 수정 하고, 전환 효과를 수정 하고, 전환 중 재생할 소리를 지정할 수 있습니다. 또한 다음으로 진행 하기 전에 슬라이드에 소요 되는 시간을 지정할 수 있습니다.

[애니메이션]
1) 미리보기
 
지정한 애니메이션을 미리 볼 수 있습니다.

2) 애니메이션
 
애니메이션의 스타일을 변경할 수 있습니다. 또한 효과를 변경이 가능합니다.

3) 고급 애니메이션
 
지정한 애니메이션의 효과를 좀더 디테일 하게 조정하거나 기존의 애니메이션 개체에 추가로 애니메이션을 추가할 수 있습니다.

4) 타이밍
 
지정한 애니메이션의 시작 속도와 재생 시간 시작 지연 시간 등을 조절하고 애니메이션 간의 순서를 변경할 수 있습니다.

[슬라이드 쇼]

슬라이드 쇼 시작
 
 슬라이드의 처음부터 시작하기 위해서는 처음부터, 현재 보고있는 슬라이드부터 시작하려면 현재 슬라이드 부터를 사용합니다. 

온라인 프레젠테이션
 
 인터넷을 통해 원격 사용자에 게 PowerPoint 프레젠테이션을 브로드캐스트할 수 있습니다. PowerPoint 내에서 슬라이드 쇼를 발표 하는 동안 청중의 구성원이 웹 브라우저에서 팔로우 합니다.

[슬라이드 쇼 재구성]
 
 재구성 한 쇼를 만들 때 다른 청중들을 위해 프레젠테이션을 조정할 수 있습니다. 재구성한 쇼를 사용하면 프레젠테이션에서 특정 슬라이드만 발표하거나 프레젠테이션의 특정 슬라이드 그룹에 연결되는 하이퍼링크를 만들 수 있습니다.

[슬라이드 쇼 설정]
 
 슬라이드 쇼 설정을 통해 보기 형식, 보기 옵션, 슬라이드 표시, 화면 전환, 슬라이드 쇼 모니터를 설정할 수 있습니다.

[슬라이드 숨기기]
 
 프레젠테이션 중에 현재 슬라이드가 보이지 않게 숨길 수 있습니다.

[예행 연습]
 
  프레젠테이션을 청중에게 제시하기 전에 프레젠테이션을 녹음하고 시간을 기록하는 데 도움이 됩니다.

[현재 슬라이드 녹화]
 
 설명, 잉크, 레이저 포인터 동작, 재생을 위한 슬라이드 및 애니메이션 시간을 기록합니다.

[모니터]
 
 슬라이드 쇼에 사용할 모니터를 선택합니다.

	Flow-Chart 정리
	개념 정리
	1.	정의

- 워크플로 혹은 프로세스를 보여주는 다이어그램의 한 종류

- 여러 종류의 상자와 이를 이어주는 화살표를 이용하여 주어진 문제에 대
한 솔루션 모델을 보여준다.

- 작성시 필요한 기호는 국제 표준화 기구에서 표준안으로 결의된 기호를 사용

2.	사용규칙

1) 기호의 내부에는 처리해야 할 내용이 들어가야 한다.

2) 순서는 위에서 아래로, 왼쪽에서 오른쪽을 원칙으로 하며 그 외의 경우는 화살표를 사용해야 한다.

3) 흐름선은 서로 교차해도 무관하며 서로 영향을 주지 않는다.

4) 흐름선 여러 개가 모여 하나로 합칠 수 있다.

5) 기호의 모형은 가로, 세로의 비율은 정하지 않으나 잘 구분할 수 있어야 한다
3.	종류

[시스템 순서도]
- 어떤 작업을 하는지 알려주는 순서도
- 자료가 정보로 변환되는 과정을 컴퓨터가 처리하는 과정, 데이터의 흐름을 중심으로 도식화한 것
- 데이터 흐름 중심 작성이므로 데이터가 처리되는 작업 단위로 나타내고 데이터가 변환되는 매체들 표현
- 프로그램 논리 작성 안함
- 작업 내용 총괄 검토 및 프로그래밍 작업과 연결시 사용

-순서도의 설명의 세밀도에 따라 개략 순서도와 상세 순서도로 나뉨
-프로그램 작성시 일반적으로 사용

2-1) 개략 순서도
    - 하나의 업무를 전체적, 종합적으로 나타낸 순서도
    - 해당 작업의 진행 순서 표시
    - 프로그램 작성시 개략 순서도를 작성하고 논리적으로 이상이 없는지 검토하게 되면 이어 상세 순서도 작성

문제 : 원형이「void PrintString(char* pszData);」와 같은 함수를 작성합니다. 이 함수는 인자 로 전달받은 문자열을 화면에 출력합니다. 최초에 호출될 때는 길이와 상관 없이 무조건 출력하지만 이후로는 앞서 출력한 문자열보다 인자로 전달받은 문자열이더길 경우에만 출력합니다. main( ) 함수의 코드가 다음과 같다면 Test와 String만 화면에 출력되어야 합니다. 단, 절대로 전역 변수를 사용하지 말아야 합니다.

#define _CRT_SECURE_NO_WARNINGS
#include<stdio.h>
#include<stdlib.h>
#include<string.h>

void PrintString(char* pszData);

int main() {
	char* str = malloc(sizeof(char) * 100);
	while (1) {
		puts("문자열을 입력해주세요.");
		scanf("%s", str);
		if (strlen(str) == 1 || str[0] == 'q')
			return 0;
		PrintString(str);
	}
}
void PrintString(char* pszData) {
	static int before_len = 0;

	int now_len = strlen(pszData);

	if (now_len > before_len) {
		puts("입력된 문자열이 더 깁니다.");
		puts(pszData);
		before_len = now_len;
		printf("%d문자이상", before_len);
	}
	else
		puts("입력된 문자열이 이전 것보다 짧아서 출력하지 않습니다");
	return 0;

}
